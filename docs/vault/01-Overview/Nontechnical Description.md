# A non‑technical description of OME

Open Educational Resource (OER) libraries have created a wealth of
educational content — lesson plans, datasets, worksheets, images, videos —
but each platform has its own editing tools and its own standards for
subject classification, grade level, author attribution, and licensing.
The result is *islands of innovation*: plenty of great content, but hard
to discover or share across systems.

**The Open Metadata Exchange is a set of bridges between those islands.**

OME gives each library a common way to *describe* its resources so any
other library can *find*, *subscribe* to, or *import* them. No library is
forced to publish; those that do become discoverable everywhere.

## Why decentralised?

OER projects lose funding. Organisations reorganise. Domains expire. When
the originating site disappears, its resources usually disappear with it.

Because OME replicates metadata across many participating servers, a
resource remains discoverable even after its original home has shut down.
Think of it as *mirroring the card catalogue*, not the books themselves —
but the card catalogue is enough to find an archived copy, locate the
authors, or reconstruct the work.

## Why scalable?

The network grows by peers joining it. Each new library that publishes
metadata adds value for every existing library. There is no single service
to scale, no bottleneck to upgrade. This is the same mathematical shape as
email or the web.
